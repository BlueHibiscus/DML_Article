Understanding DML Commands of SQL Server with Examples

What is SQL?

Structured Query Language (SQL) is a programming language developed in the 1970s by IBM researchers. It enables creation and manipulation of databases and uses certain commands to carry out the operations designed to facilitate the interaction between databases. SQL statements are classified in various ways which are grouped into sublanguages. The term “Sublanguage” was first brought out by E.F. Codd in 1970 that refers to a computer language used to define or manipulate the structure and contents of a Relational database management system (RDBMS). Sublanguages in SQL is a fascinating technology. Once you get a hang of it, you will no doubt see its value. I, however, in this brief article am specifically picking the DDL aspect of the Sub-language, hope this makes your understanding easier with the examples used.

DML (DATA MANIPULATION LANGUAGE): It is a computer language consisting of SQL commands that are used are used to insert and modify, the contents of tables in a database. Insert, Delete, Update, and Merge are DML commands.

Insert:

This command makes inserting new data into the table very easy. All you need to do is specify into which table you want to insert data, into which columns to insert data, and finally what data to insert.

Note: Inserting text in a column with Character data types require single quote (‘).

Inserting one or more rows in all the columns or selective column/s.

Ex 1: To insert records in all the columns. Let’s go step by step here.

Step1: Create a table EMPLOYEE in the database SQLSTUDIES with three column EMPID, ENAME, and SALARY.

Step2: Inserting records in all the columns

Syntax: INSERT [INTO] table_ name VALUES (data), (data), (data)

Step3: Inserting records in all the column using SELECT command

Note: INTO keyword is Optional in SQL Server.

You can see we have inserted three records each into the EMPLOYEE table using different insert styles and both the styles are valid. Choose any style you are comfortable with.

EX2: Insert records in selected columns.

You can see to insert record in selected columns both the insert styles are valid. Mention the name of the column explicitly before inserting data into the column. As seen in the result set, the columns we have not entered the value, is a NULL value.

Ex 3: EID column is integer data type but we have entered character value in the column.

Make sure while inserting records in particular columns the data type of columns should match with the value entered, else we get an error message.

Ex 4: Creating a new table on the fly and inserting rows from existing table.

Step 1: Here a new table is created without writing “Create” command and all rows are automatically populated into the new table. This is same as creating a duplicate table.

Syntax: SELECT * INTO [TABLE_NAME2] FROM [TABLE_NAME1]

You can see a new table DUPLICATE_EMPLOYEE is created from an existing table EMPLOYEE by copying the existing table column names. All records from EMPLOYEE table are also populated into the new table.

UPDATE:

This command modifies data existing in the tables.

Syntax : UPDATE SET = value .... [WHERE {condition}]

Ex1: Update salary from 20000 to 25000 of Employee whose EMPID is 1.

After running the UPDATE command, the Salary has updated. The where keyword is used for selecting the record.

Ex 2: Update the Salary to 15000 where Salary is NULL.

A field with a NULL value is a field with no value.

NOTE: A NULL value is different than a zero value or a field that contains spaces.

Ex 3: Update all records of Salary column.

in this example you can see WHERE clause is not used since all the rows in Salary column are updated.

DELETE:

This command deletes one or more records.

Syntax: DELETE FROM [WHERE ]

Ex1 : Delete all records from EMPLOYEE table.

Ex2 : Deleting selective records. Deleting from EMPLOYEE table where EMPID = 1

SUMMARY:

SUMMARY: This article completes introduction to DML commands for the beginners in SQL Server step by step including various simple examples to show how the DML syntaxes are executed.

• It explains how to add new records in a table using INSERT command in different styles. To add a new record, you must specify which table the record should go into, which fields you assign values to, and finally the values to be assigned.

• Update command modifies the data already in the table. WHERE filter specifies which records in the table receives updates.

• Delete command deletes all the records from the table. WHERE clauses specifies which records you want to delete.

Happy learning.
